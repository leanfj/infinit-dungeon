# Repository Guidelines

## Project Structure & Module Organization
The project is organized into distinct modules to enhance maintainability and scalability. Each module resides in its own directory, encapsulating related functionalities. Below is an overview of the primary directories:
- `/entities`: Contains all game entities, including players, enemies, and interactive objects.
- `/HUD`: Houses the heads-up display components and scripts.
- `/docs`: Dedicated to project documentation, including implementation details and guidelines.
- `/scenes`: Contains the various game scenes and levels.
- `/assets`: Stores all game assets such as images, sounds, and animations.
- `/mechanics`: Contains scripts and resources related to game mechanics and systems.
- `/terrain`: Includes terrain generation and management scripts.

## Coding Style & Naming Conventions
- Follow Godot's GDScript style guide for consistency.
- Use descriptive names for variables, functions, and classes.
- Maintain consistent indentation (4 spaces).
- Comment code where necessary to explain complex logic.

## Documentation

Todo arquivo de documentação deve ser salvo na pasta `/docs`
Sempre que finalizar a task solicitada crie a documentação sobre o que foi realizado com o arquivono formado `descricao_implementacao_timestamp.md` onde em `descricao_implementacao` dever ser algo relacionado ao que foi feito exmeplo `REFATORACAO_HOME.md`.
